Results and Discussions

  • Programming language advantages 
    • Programming languages are formal languages that are invented and designed in order to communicate the instructions to the machine especially the computer, they can also be used in creating programs which are responsible for controlling the machine's behavior or expressing algorithms.

    • The programming languages of computers are the languages that are used in writing computer programs that include the performing of the computer in the sort of computation or algorithm, they can also be used in controlling the external devices of the computer such as printers, disk drives & robots.

    • If you understand programming Language, You can increase your salary especially if you can build a website or make a good application, programming is very easy once you know how to syntax the programming language, you will also get many new showcases in your career.

    • Codes of programming are the instructions used to define the computer tasks. Learning and using programming languages are easy as you can create websites of high quality. The beginning of all the websites and the applications is by a little code and then developed by time.

  • Programming language disadvantages
    • Annoying bugs may appear where solving them is difficult sometimes. It is not easy to be a programmer if your math skills are not good as you have to think a lot. Memorizing the syntax of the programming language will be difficult too if your memory is not good.

    • If you are slow in typing the programming language, you will consume a lot of time in order to program something. If you already know the syntax you will take seconds to type a lot of codes, but this skill needs thinking and logic as some programming languages are difficult in there usage.

    • "C" programming language doesn't contain the concept of OOPs, that’s why programmers developed "C++". Checking of run time isn't included in "C" language, there is no definite method to check, an integer value can be passed, it is used in the type of floating data, it doesn’t include the namespace concept, it neither include the concept of constructor nor destructor.

    • It is difficult to apply code from one program to another, if you want to modify a part of the code you have to modify the whole code. As the code develops, it gets more complicated and it becomes difficult to understand or modify it. The functional Languages aren't qualified to do IO, They aren't able to communicate with the outside world, and you can't be completely sure that first line of code will be executed before second.
